MCPを巡る議論に新展開
軽量版MCPとAgent Skillsの組み合わせが最適解として浮上
EXHIBIT 7
MCP vs Agent Skillsの二項対立から「レイヤード・アーキテクチャ」へ ― Skills＝知識層、MCP＝実行層の組み合わせが主流に
MCP と Agent Skills の比較
項目 MCP Agent Skills
役割 外部ツール・データとの標準通信プロトコル タスク実行の手順・知識をファイルとしてパッケージ化
提供元 Anthropic発、Linux Foundation管理 各社独自（Anthropic Skills, MS等）
構成要素 Resources, Prompts, Tools SKILL.md＋テンプレート＋参照文書
強み N×M統合問題を解決、標準化 軽量、ノーコード、メタデータ省トークン
課題 トークンコスト大（例：Sentry MCP〜8Kトークン） 単独では外部ツール接続不可
核心的見解：「SkillsがMCPを置き換える」は誤り。Skillsはワークフローを記述し、MCPはその実行基盤を提供する。料理に例えれば、Skillsはレシピ、MCPは調理器具。両者の組み合わせで初めてエージェントは効果的に動作する。
軽量化と組み合わせの具体策
SkillPort（新ツール）
Anthropicの「Tool Search Tool」パターンに着想。メタデータのみ先読み（〜100トークン/Skill）→必要時に詳細をオンデマンドロード。50以上のSkillsでもコンテキスト圧迫なし。Cursor, Copilot, Windsurf等に対応。
MCPのコード実行最適化
Anthropic公式ガイダンスとして、ツール定義をファイルシステム上のコードとして公開し、オンデマンド読み込みで効率化。データのフィルタリングをMCPサーバー側で実行し、モデルへのトークン転送を最小化。
ガバナンスの進化
MCPはLinux FoundationのAgentic AI Foundation（AAIF）に寄贈済。Anthropic、Block、OpenAIが共同設立。OAuth Resource Server、RFC 8707準拠のセキュリティ強化が2026年仕様に追加。
Source: cra.mr, LlamaIndex, Armin Ronacher, Block/Goose, Anthropic Engineering（2025-2026年）
